# lhw-book2doc

## 目标

把书籍和长文档转换成一个 AI 易于理解的单文件文档。重点不是粗暴抽文本，而是尽量保留书的结构：书名、目录、章节顺序、标题层级、段落、脚注、引用和参考文献；同时清理阅读器噪音，比如图片占位符、EPUB 内部锚点、页眉页脚、断裂链接和残留 HTML 标签。

默认输出 Markdown：`<书名>.ai.md`。Markdown 通常比纯文本更适合 AI，因为标题、列表、引用、脚注和段落边界都能保留下来。只有在用户明确要求纯文本，或原文没有可靠结构时，才输出 `.ai.txt`。

## 快速使用

优先使用 skill 自带的 JS 脚本：

```bash
/Users/hwo/CodeSpace/lhw-skills/lhw-book2doc/scripts/book2doc.js "/path/to/book.epub"
```

默认把结果输出到源文件所在目录。可选参数：

```bash
book2doc.js 输入文件 [输出文件] [md|txt]
```

示例：

```bash
book2doc.js "book.epub"
book2doc.js "scan.pdf" "scan.ai.md" md
book2doc.js "book.pdf" "book.ai.txt" txt
```

## 格式策略

- **EPUB / HTML / DOCX / RTF / ODT / Markdown**：优先用 `pandoc` 转成 Markdown。
- **文本型 PDF**：优先用 `pdftotext -layout` 抽取文本，再包装成 Markdown；没有 `pdftotext` 时尝试 `pandoc`。
- **扫描版 PDF / 纯图片 PDF**：先尝试普通文本抽取；如果抽出的文字太少，判定为需要 OCR。
- **MOBI / AZW3 / FB2**：如果安装了 Calibre 的 `ebook-convert`，先转 EPUB，再转 Markdown；没有就明确告诉用户缺少依赖。
- **TXT**：直接规范化成 `.ai.md` 或 `.ai.txt`。

## 扫描 PDF 处理

不要因为生成了文件就假设 PDF 转换成功。PDF 抽取后必须检查文本量：

- 一本多页书抽出的中英文字符少于约 800 个，通常说明它是扫描版或图片型 PDF；
- 大量重复页码、页眉、页脚，但正文很少，也说明抽取失败；
- 页面几乎为空，或只有标题/页码，说明需要 OCR。

OCR 优先级：

1. 如果有 `ocrmypdf`：先生成带文字层的临时 OCR PDF，再抽文本。
2. 如果有 `tesseract` 和 `pdftoppm`：先把 PDF 页面渲染成图片，再逐页 OCR。
3. 如果本机有 macOS Vision 或其他 OCR 流程，也可以使用。
4. 如果没有 OCR 工具，停止并清楚告诉用户缺少 OCR 能力；不要把不完整结果说成转换完成。

中文书优先使用 `chi_sim`、`chi_tra`、`eng` 等 OCR 语言包。

## 清理原则

“噪音”不是固定概念。清理前先判断：它是源书内容，还是转换过程产生的机器残留？宁可保留一点可疑内容，也不要删掉可能有语义价值的信息。

默认只清理高度确定的技术残留；对图注、脚注、引用、表格说明、图片 alt 文本等可能承载信息的内容，要保留或转写成普通文本。

## 清理规则

转换后清理明显噪音：

- 空图片占位：`![](...)`、`![alt](...)`、`[alt]`、空 `[]`；
- EPUB 内部结构：`<span id=...>`、`part0001.xhtml`、内部跳转锚点；
- 残留 HTML 容器标签：`div`、`p`、`section` 等；
- 重复空行；
- 明显重复的页眉、页脚、页码；
- 本地图片路径，如 `Images/cover.jpeg`。

保留有意义内容：

- 非空且有描述性的图片 alt 文本，转写成“图像说明：...”；
- 图片说明、图注和表注；
- 脚注、尾注、引用、参考文献；
- 有助于导航的目录；
- 作者简介、译者说明、出版说明等属于书籍内容的部分。

不要无脑删除：

- 普通网页链接，除非它只是 EPUB 内部跳转；
- 表格内容，即使排版变差；
- 数学、代码、术语中使用的尖括号内容；
- 章节标题附近的短行，它们常常是结构而不是噪音。

## 验证清单

完成前必须检查：

1. 输出结果存在，并且是单个文件。
2. 文件大小和原书规模大致匹配。
3. 前 80 行是可读文本，而不是标签、图片路径或占位符。
4. 如果源书有章节，输出中应能看到章节或标题线索。
5. 搜索常见残留：`[alt]`、`![](`、`<span`、`<a`、`part000`、`Images/`。
6. 如果来源是 PDF，要说明它是文本抽取还是 OCR 抽取。

如果验证失败，继续清理或明确报告阻塞点。

## 交付说明

回复用户时说明：

- 输出文件路径；
- 为什么选择这个格式；
- PDF 是否需要 OCR；
- 已知限制，例如缺少 OCR 工具、章节标题不稳定、表格可能需要人工复查等。
